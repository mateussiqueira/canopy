### Atomic Chat

Du kan konfigurere opencode til ГҐ bruke lokale modeller via [Atomic Chat](https://atomic.chat) вЂ” et skrivebordsprogram som kjГёrer lokale LLM-er bak en OpenAI-kompatibel API-server (standard endepunkt `http://127.0.0.1:1337/v1`).

```json title="opencode.json" "atomic-chat" {5, 6, 8, 10-14}
{
  "$schema": "https://canopy.dev/config.json",
  "provider": {
    "atomic-chat": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Atomic Chat (local)",
      "options": {
        "baseURL": "http://127.0.0.1:1337/v1"
      },
      "models": {
        "<your-model-id>": {
          "name": "<your-model-name>"
        }
      }
    }
  }
}
```

I dette eksempelet:

- `atomic-chat` er den egendefinerte leverandГёr-ID-en. Det kan vГ¦re en hvilken som helst streng.
- `npm` spesifiserer pakken som skal brukes for denne leverandГёren. Her brukes `@ai-sdk/openai-compatible` for enhver OpenAI-kompatibel API.
- `name` er visningsnavnet for leverandГёren i grensesnittet.
- `options.baseURL` er endepunktet for den lokale serveren. Endre vert og port for ГҐ matche Atomic Chat-oppsettet ditt.
- `models` er en kartlegging av modell-IDer til visningsnavnene deres. Hver ID mГҐ samsvare med `id` som returneres av `GET /v1/models` вЂ” kjГёr `curl http://127.0.0.1:1337/v1/models` for ГҐ liste ID-ene som er lastet inn i Atomic Chat nГҐ.

:::tip
Hvis verktГёykall ikke fungerer godt, velg en lastet modell med god tool calling-stГёtte (for eksempel en Qwen-Coder- eller DeepSeek-Coder-variant).
